Overview

This guide explains the GitHub Spec Commands (GSC) enhancement system with memory integration, performance targets, and cross-platform support. It follows documentation best practices—clear structure, readable language, and task-oriented organization.

See also gsc-enhancement-system.md. Use the search box to filter commands.


Quick Start

  1. Plan: clarify goals and users; create or review plan.md and tasks.md.
  2. Execute: run spec-driven workflow (see Workflows below).
  3. Validate: run quality gates; review performance and memory checks.
  4. Recover: on failures, use rollback and rerun implement.

Commands

Each command includes a description, usage, optional parameters, notes, and examples.

Workflows

  • Spec-Driven: constitution → specify → clarify → plan → task → analyze → implement → validate → status
  • Error Recovery: validate → status → rollback (--full-reset optional) → implement
  • Cross-Platform: use PowerShell Core 7+ or shell wrappers at .specify/scripts/gsc/*.sh

Memory Integration

Memory files guide generation and validation. Access is logged and protected. Locations are auto-detected using .specify/config/memory-paths.json. Performance targets aim for total processing under 5 seconds.

  • Security: checksum validation, optional encryption, and access logging
  • Performance: cached reads; pattern extraction targets < 3s
  • Location: per-OS base directory and filenames defined via configuration

Troubleshooting

  • Use ./gsc help for command JSON and examples.
  • Check .specify/state/gsc-workflow.json for current phase and state.
  • If a step fails, run ./gsc rollback --full-reset (optional) and then rerun implement.
  • Confirm PowerShell Core 7+ availability; use shell wrappers for macOS/Linux.

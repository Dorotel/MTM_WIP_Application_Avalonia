This page provides an overview of the GitHub Spec Commands (GSC) enhancement system with memory integration, performance targets, and cross-platform support. See also gsc-enhancement-system.md.

Tip: use the search box to filter commands.

Commands

Workflows

  • Spec-Driven: constitution → specify → clarify → plan → task → analyze → implement → validate → status
  • Error Recovery: validate → status → rollback (-FullReset optional) → implement
  • Cross-Platform: use PowerShell Core 7+ or shell wrappers under .specify/scripts/gsc/*.sh

Memory Integration

Memory files guide generation and validation. Access is logged and protected.

  • Security: checksum-validator.ps1, memory-encryption.ps1, access-logger.ps1
  • Performance: memory-processing targets < 5s
  • Location: auto-detected via memory-paths.json

Troubleshooting

  • Use ./gsc help for command JSON and examples
  • Check .specify/state/gsc-workflow.json for current phase
  • On failures, run ./gsc rollback -FullReset then re-run implement

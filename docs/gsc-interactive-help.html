Complete user manual for manufacturing-grade development workflows with memory integration, cross-platform support, and 24/7 operational reliability.

Overview

What is GSC?

The GitHub Spec Commands (GSC) Enhancement System is a manufacturing-grade development automation platform designed for 24/7 operational environments. It integrates memory-driven patterns, cross-platform workflows, and comprehensive manufacturing domain knowledge to accelerate software development while maintaining enterprise-grade reliability.

🏭 Manufacturing-Grade

Built for 24/7 operations with robust error recovery, performance monitoring, and systematic validation gates.

🧠 Memory Integration

Learns from past development patterns across domains: Avalonia UI, debugging workflows, custom controls, and universal patterns.

🎯 Spec-Driven

Follows systematic specification-driven workflow from requirements capture through implementation and validation.

Key Capabilities

  • Memory-Driven Development: Integrates 4+ memory domains (avalonia-ui-memory, debugging-memory, avalonia-custom-controls-memory, universal memory)
  • Manufacturing Context: Deep integration with MTM inventory systems, part validation (PartID/Operation/Location patterns), and manufacturing workflows
  • Cross-Platform Support: PowerShell Core 7.0+ with shell wrappers for macOS/Linux, performance targets <5 seconds
  • Enterprise Security: Checksum validation, optional encryption, access logging, and protected memory file operations
  • Performance Monitoring: Real-time degradation detection, workflow state tracking, and rollback capabilities
Documentation Structure: This manual follows Diátaxis framework principles - Tutorials for learning, How-to guides for problems, Reference for information, and Explanations for understanding. Navigate using the sidebar for task-oriented access.

Installation

Prerequisites

Component Version Status Notes
PowerShell Core 7.0+ Required Cross-platform shell execution
.NET Runtime 8.0+ Ready For MTM application context
Git 2.30+ Ready Version control integration
VS Code 1.80+ Optional Enhanced development experience

Setup Steps

  1. Verify PowerShell Core: pwsh --version should return 7.0 or higher
  2. Initialize GSC Structure: Navigate to your project root and ensure .specify/ directory exists
  3. Configure Memory Paths: GSC auto-detects memory file locations using .specify/config/memory-paths.json
  4. Validate Installation: Run ./gsc help to confirm all components are accessible
  5. Test Workflow: Execute ./gsc status to verify system readiness

Verification Commands

# Check PowerShell version
pwsh --version

# Verify GSC accessibility
./gsc help

# Check system status
./gsc status --performance

Quick Start Tutorial

Your First GSC Workflow

This tutorial walks you through creating a simple inventory feature using GSC's spec-driven workflow. You'll learn the core commands and see how memory integration enhances development.

Learning Objective: Complete a full development cycle from specification to implementation using GSC automation with manufacturing domain patterns.

Step-by-Step Walkthrough

  1. Create Feature Specification
    ./gsc specify "Add inventory status panel for real-time part tracking"

    GSC will generate a comprehensive specification using Avalonia UI memory patterns and manufacturing domain knowledge.

  2. Clarify Requirements
    ./gsc clarify

    Interactive guided questions help refine requirements using debugging memory patterns.

  3. Generate Technical Plan
    ./gsc plan

    Creates plan.md with architecture decisions and implementation approach.

  4. Create Task Breakdown
    ./gsc task

    Generates tasks.md with ordered, trackable implementation tasks.

  5. Analyze Current State
    ./gsc analyze

    Analyzes repository structure and validates readiness for implementation.

  6. Execute Implementation
    ./gsc implement

    Performs automated implementation following MTM patterns and memory-driven guidance.

  7. Validate Results
    ./gsc validate

    Runs comprehensive validation including constitutional compliance and manufacturing gates.

  8. Check Status
    ./gsc status --performance

    Reviews workflow completion and performance metrics.

Understanding the Workflow States

GSC maintains workflow state in .specify/state/gsc-workflow.json and provides recovery mechanisms at each phase.

Phase Purpose Outputs Recovery
specify Feature specification creation spec.md with requirements ./gsc rollback
clarify Requirements refinement Updated spec.md sections ./gsc rollback
plan Technical architecture plan.md with decisions ./gsc rollback
task Implementation breakdown tasks.md with ordered steps ./gsc rollback
implement Code generation/editing Modified source files ./gsc rollback --full-reset
validate Quality gate checks Validation reports Fix issues, re-validate

Common First-Time Issues

PowerShell Execution Policy: If you encounter execution policy errors, run Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Memory File Access: First run may be slower as GSC discovers and indexes memory files. Subsequent runs benefit from caching.
Performance Expectations: Initial workflow complete should finish under 5 seconds total processing time. Use --performance flag to monitor.

GSC includes deep integration with MTM (Material Handler Manufacturing) systems, understanding inventory operations, part validation patterns, and manufacturing business rules.

Automated workflows for inventory movement validation, including source/destination verification and quantity checking.

Quality gates specific to manufacturing operations, including business rule validation and operational compliance.

Memory Management

Memory Integration System

GSC integrates with four specialized memory domains that capture development patterns, troubleshooting approaches, and domain-specific knowledge.

Memory Domain Focus Area Key Patterns Update Trigger
avalonia-ui-memory Avalonia UI Framework MVVM binding, layout constraints, container hierarchy UI-related implementations
debugging-memory Systematic Debugging Root cause analysis, evidence-based fixing, troubleshooting workflows Bug fixes and diagnostics
avalonia-custom-controls-memory Custom Control Development Manufacturing field controls, styling systems, layout patterns Custom control creation
memory Universal Patterns Cross-project development, container management, responsive design General development patterns

Memory Operations

View Current Memory Status

./gsc memory

Shows memory file locations, integrity status, and last update timestamps.

Update Specific Memory Domain

./gsc memory --update avalonia-ui-memory --pattern "DataGrid column binding resolution"

Updates specified memory domain with new patterns from recent development experience.

Memory-Only Validation

./gsc validate --memory-only

Validates current specification against memory patterns without full workflow execution.

Memory Security: All memory operations include checksum validation, optional encryption, and access logging. Memory files are automatically discovered using configuration in .specify/config/memory-paths.json.

Troubleshooting

Common Issues and Solutions

Workflow Stuck or Failed

Symptoms: Commands hang, validation failures, or corrupted workflow state.

Solution:

./gsc rollback --full-reset ./gsc implement

Full reset returns to start of workflow with preserved memory state.

Performance Degradation

Symptoms: Commands take longer than 5 seconds, system shows degradation mode.

Solution:

./gsc status --performance

Check performance metrics and consider memory cleanup or system restart.

Memory File Access Issues

Symptoms: Memory integration failures, file not found errors.

Solution:

./gsc memory

Verify memory file locations and permissions. Check .specify/config/memory-paths.json.

Cross-Platform Issues

macOS/Linux: Use shell wrappers at .specify/scripts/gsc/ if PowerShell Core unavailable.

Windows: Ensure PowerShell Core 7.0+ is installed and execution policy allows script execution.

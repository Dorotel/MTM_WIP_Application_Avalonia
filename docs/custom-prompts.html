💡Custom Copilot Prompts

See Personas for complete persona definitions and behavioral guidelines.
▶Prompt Types
  • Create UI Element — For generating Avalonia controls or views based on mapping and instructions.
  • Create Error System Placeholder — For scaffolding a new error handling class or module with standard conventions.
  • Create Logging Info Placeholder — For generating a logging helper or logging configuration according to project standards.
  • Create Business Logic Handler — For scaffolding a business logic class or method with correct naming and separation from UI.
  • Create Database Access Layer Placeholder — For generating a stub or scaffolding for database interaction classes or repositories.
  • Create Unit Test Skeleton — For generating a basic, empty unit test class or method in the appropriate structure.
  • Create Configuration File Placeholder — For generating an empty config/settings file with standard project structure.
  • Refactor Code to Naming Convention — For requesting Copilot to rename code elements to fit project naming rules.
  • Add Event Handler Stub — For quickly generating empty event handler methods (with TODOs) for specific controls.
  • Document Public API/Class — For generating XML documentation for public members.
  • Create Customized Prompt — For drafting new actionable custom prompts and assigning appropriate personas.
▶Rule for Adding Custom Prompts
  • Whenever a new custom prompt is added to this file, an example usage for that prompt must also be added to Prompt Examples.
  • For persona descriptions, see Personas.

▶MTM-Specific Workflow Patterns

UI Generation from Markdown Files

  1. Parse Component Hierarchy ‚Äî Extract the tree structure and convert to Avalonia AXAML
  2. Map MTM Data Types ‚Äî Use Part ID (string), Operation (string numbers), Quantity (integer)
  3. Apply Context Menu Patterns ‚Äî Add management features via right-click menus
  4. Implement Space Optimization ‚Äî Use UniformGrid and proper alignment for component removal
  5. Follow MTM Color Scheme ‚Äî Apply purple brand colors consistently
  6. Create ReactiveUI Bindings ‚Äî Generate ViewModels with proper observable patterns

Quality Assurance Workflow

  1. Initial Assessment ‚Äî Review code files against all instruction guidelines
  2. Violation Identification ‚Äî Document specific non-compliance issues with examples
  3. Priority Classification ‚Äî Categorize issues as High/Medium/Low priority
  4. Fix Recommendations ‚Äî Provide specific guidance for bringing code into compliance
  5. Report Generation ‚Äî Create standalone compliance reports in Compliance Reports/ folder
  6. Re-verification ‚Äî Follow up reviews to confirm compliance improvements

Missing Systems Implementation Workflow

  1. Foundation Phase ‚Äî Implement data models, Result pattern, and DI container (Prompts 21-24)
  2. Service Layer Phase ‚Äî Create all service interfaces and implementations (Prompts 25-28)
  3. Infrastructure Phase ‚Äî Add navigation, themes, repositories, and validation (Prompts 29-32)
  4. Quality Assurance Phase ‚Äî Implement testing, logging, caching, and security (Prompts 33-36)
▶Persona Behavioral Guidelines

MTM-Specific Behaviors

  • Operations are Numbers ‚Äî Always treat operations as string numbers ("90", "100", "110"), not action descriptions
  • Part ID Format ‚Äî Use string format for Part IDs (e.g., "PART001")
  • 1-Based Indexing ‚Äî UI positions use 1-based indexing for display
  • Purple Brand Consistency ‚Äî Apply MTM purple color scheme (#4B45ED primary, #BA45ED accents)
  • ReactiveUI Patterns ‚Äî Use ReactiveObject, RaiseAndSetIfChanged, ReactiveCommand, WhenAnyValue
  • Modern Layout Principles ‚Äî Prefer Grid over StackPanel, use cards and sidebars
  • Context Menu Management ‚Äî Add right-click functionality for component management
  • Business Logic Separation ‚Äî Leave implementation as TODO comments, focus on structure

Quality Assurance Behaviors

  • Comprehensive Review ‚Äî Check all aspects of code against instruction guidelines
  • Structured Reporting ‚Äî Use standardized format for violation documentation
  • Priority Assignment ‚Äî Classify issues based on impact and compliance severity
  • Solution-Oriented ‚Äî Provide specific fix recommendations with instruction references
  • Progress Tracking ‚Äî Update compliance status and maintain audit trail

Missing Systems Implementation Behaviors

  • Foundation First ‚Äî Always implement foundational systems before dependent systems
  • Clean Architecture ‚Äî Follow dependency inversion and separation of concerns
  • Error Handling ‚Äî Integrate Service_ErrorHandler in all new implementations
  • Testing Support ‚Äî Include testability considerations in all implementations
  • Documentation ‚Äî Provide comprehensive XML documentation for all public APIs
  • DI Preparation ‚Äî Ensure all services are prepared for dependency injection

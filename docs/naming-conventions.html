📝Naming Conventions

📌 Note: These naming conventions apply to the missing core systems that need to be implemented. Verify compliance using Quality Assurance reports.
▶Naming Rules
  • Files: Use PascalCase for all file names. Examples:
    • MainView.axaml
    • InventoryViewModel.cs
    • IInventoryService.cs (interface)
    • InventoryService.cs (implementation)
    • Control_ErrorMessage.axaml (control prefix)
  • Classes: Use PascalCase. Examples:
    • MainView
    • InventoryViewModel
    • InventoryService
    • Result<T>
  • Interfaces: Prefix with "I". Examples:
    • IInventoryService
    • INavigationService
    • IApplicationStateService
  • Methods: Use PascalCase and async suffix. Examples:
    • GetInventoryAsync()
    • SaveInventoryItemAsync()
    • ProcessOperationAsync()
  • Properties: Use PascalCase. Examples:
    • PartId (string)
    • Operation (string numbers like "90", "100")
    • Quantity (integer)
    • IsLoading (boolean)
  • Fields: Use _camelCase for private fields. Examples:
    • _partId
    • _inventoryService
    • _cancellationTokenSource
▶MTM-Specific Naming
  • Part IDs: Use "PartId" (not "PartNumber" or "ID")
  • Operations: Use "Operation" for string numbers ("90", "100", "110")
  • Quantities: Use "Quantity" (not "Qty" or "Amount")
  • User References: Use "UserId" for string user identifiers
  • Timestamps: Use "CreatedDate", "ModifiedDate" (not "Created", "Modified")
  • Status Fields: Use descriptive names like "ConnectionStatus", "ProcessingStatus"
▶Service Naming Patterns
  • Interface Pattern: I{Domain}Service
  • Implementation Pattern: {Domain}Service
  • Repository Pattern: I{Entity}Repository, {Entity}Repository
  • Data Transfer Objects: {Entity}Dto
  • Request/Response: {Operation}Request, {Operation}Response

Examples

  • IInventoryService, InventoryService
  • IInventoryRepository, InventoryRepository
  • InventoryItemDto
  • GetInventoryRequest, GetInventoryResponse
▶Command and Property Naming
  • ReactiveCommands: Use action + "Command" suffix
  • Observable Properties: Use descriptive property names
  • Collections: Use plural nouns
  • Boolean Properties: Use "Is", "Has", "Can" prefixes

Examples

  • SaveCommand, LoadDataCommand, RefreshCommand
  • SelectedInventoryItem, FilterText, CurrentUser
  • InventoryItems, TransactionHistory, QuickButtons
  • IsLoading, HasErrors, CanSave

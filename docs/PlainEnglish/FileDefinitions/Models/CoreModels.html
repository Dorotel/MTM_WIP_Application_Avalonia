üè¢ MTM-Specific Business Conventions

üö® Critical MTM Data Conventions:
  • Part IDs: Always strings (e.g., "PART001", "COMPONENT-X")
  • Operations: String numbers representing workflow steps (e.g., "90", "100", "110")
  • UI Positions: 1-based indexing for display (Position 1-10, not 0-9)
  • TransactionType: Determined by user intent, NOT operation numbers
  • Compatibility: Property aliases for legacy code integration

🤔 What Does This Do?

Think of the UserManagementViewModel as the HR office for the factory floor - it's where supervisors and managers control who can use the WIP system and what they're allowed to do. This is like having a security badge system that determines which doors each worker can open.

This screen allows authorized managers to:

  • 👤 Add new workers to the system
  • ✏️ Update worker information (names, departments, permissions)
  • 🔒 Set access levels (what each worker can and can't do)
  • 📊 Monitor user activity (who's using the system when)
  • 🚫 Disable accounts when workers leave or change roles
  • 🔐 Reset passwords when workers need help

🏭 Real-World Manufacturing Example

Scenario: Setting Up a New Worker

Janet, the plant manager, needs to set up system access for Tom, a new inventory clerk who just started today.

1

Open User Management

Janet opens the User Management screen to add Tom to the system. Only managers have access to this administrative function.

2

Create New User Account

She clicks "Add New User" and enters Tom's information: full name, employee ID, department (Inventory), and shift schedule.

3

Set Permission Level

Since Tom is an inventory clerk, Janet sets his role to "Inventory Worker" which allows him to add, remove, and transfer parts but not manage other users.

4

Assign Temporary Password

The system generates a temporary password that Tom will need to change when he first logs in.

5

Activate and Test

Janet activates Tom's account and tests it to make sure he can log in and access the inventory functions he needs for his job.

📋 MTM Business Rules

User Access Levels

  • 🔰 Basic Worker: Can add, remove, and transfer inventory only
  • 👨‍💼 Supervisor: Basic worker permissions + view reports and transaction history
  • 👩‍💼 Manager: Supervisor permissions + export data and view user activity
  • 🔧 Administrator: Full access including user management and system settings

Account Security Rules

  • 🔐 Password Policy: Passwords must be at least 8 characters with mixed case
  • 🕒 Session Timeout: Users automatically logged out after 4 hours of inactivity
  • 🚫 Account Lockout: Accounts locked after 5 failed login attempts
  • 📅 Password Expiry: Passwords must be changed every 90 days

Compliance Requirements

  • 📊 Audit Trail: All user management actions are logged
  • 🏢 HR Integration: User accounts must match current employee roster
  • 🔒 Data Privacy: Personal information protected according to company policy
  • 📋 Regular Review: User access reviewed quarterly for compliance

🔧 Common Issues & Solutions

❓ "A worker can't log in"

Solution: Check if their account is active and not locked. Reset their password if needed. Verify their employee ID is correct in the system.

❓ "Someone has wrong permissions"

Solution: Edit their user account and update their role/permissions. Changes take effect immediately.

❓ "I can't create a new user"

Solution: Make sure you have administrator permissions. Check that the employee ID doesn't already exist in the system.

❓ "The user list is missing someone"

Solution: Check your filters - they might be hiding inactive users. Use the search box to look for the specific person.

🔐 Security and Compliance Features

  • 👨‍💼 Role-Based Access: Permissions assigned by job role, not individual user
  • 🔒 Principle of Least Privilege: Users get only the minimum access needed for their job

  • 📍 Location-Based: Can limit access to specific warehouse areas or operations

Data Protection

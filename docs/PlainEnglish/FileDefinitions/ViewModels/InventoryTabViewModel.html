🎯 What This Component Does

InventoryTabViewModel is the primary inventory management interface in the MTM system. It provides a comprehensive form-based approach to inventory operations with advanced validation and automation features.

💡 Key Principle: This ViewModel uses reactive programming patterns to provide real-time validation and seamless integration with other MTM components like Quick Buttons.

📝 Form Management

Comprehensive input fields for Part ID, Operation, Location, Quantity, and Notes with real-time validation

⚡ Quick Integration

Receives input from Quick Buttons to auto-populate forms for frequently used inventory items

🔄 Smart Reset

Soft reset (clear form) or hard reset (refresh data) based on user needs and shift key detection

✅ Live Validation

Real-time form validation that enables/disables save button based on data completeness

🔄 Advanced Workflow Process

Step 1: Data Loading - System loads current parts, operations, and locations from database
Step 2: Form Population - User manually enters data or Quick Buttons auto-populate fields
Step 3: Real-Time Validation - System continuously validates input and enables/disables save button
Step 4: Save Operation - Validated data is sent to database through InventoryService
Step 5: Event Broadcasting - Success triggers events to update Quick Buttons and other components
Step 6: Form Reset - Smart reset prepares form for next entry while maintaining user context

🏢 MTM-Specific Business Rules

🚨 Critical MTM Rule: InventoryTabViewModel is operation-neutral - it can handle IN, OUT, or TRANSFER operations based on the user's intent and the specific tab context.

📊 Reactive Validation System

✅ Required Fields: Part ID, Operation, Location, Quantity 🔢 Quantity Validation: Must be positive integer 📝 Notes: Optional but tracked for audit purposes 🎛️ Save Button: Enabled only when all validations pass

🔧 Integration Patterns

  • Quick Buttons Integration: Receives QuickActionExecuted events to auto-populate forms
  • InventoryService Integration: Handles database operations through stored procedures
  • Event Broadcasting: Fires InventoryItemSaved events to update other components
  • Panel Management: Coordinates with Main View for panel visibility and layout

⚡ Advanced Features

🔄 Reactive Programming Implementation

🔍 WhenAnyValue: Real-time validation based on field changes 🛡️ ObservableAsPropertyHelper: Computed properties for CanSave state ⚡ ReactiveCommand: Commands with built-in async support 📡 Event-Driven: Loose coupling through events instead of direct references

🎛️ Reset System Options

Soft Reset: Clears form fields but keeps ComboBox data for speed
Hard Reset: Clears form AND refreshes all ComboBox data from database (Shift+Reset)

📊 Version Tracking

Displays current system version (4.6.0.0) to ensure users know which version of MTM they're working with.

üè≠ Real-World Manufacturing Analogy

Think of QuickButtonsViewModel as the "Quick Action Station" - a specialized control panel next to the main workstation that stores the 10 most frequently used operations, allowing experienced workers to execute common tasks with a single button press.

‚ö° The Quick Action Station Features:
  • 10 Memory Slots: Stores the last 10 inventory operations for instant access
  • Auto-Update System: Automatically captures successful operations and shifts the list
  • One-Click Execution: Populates main forms with pre-configured part, operation, and quantity data
  • Context Menu Management: Right-click options for editing, removing, and organizing buttons
  • Smart Positioning: New operations move to the top, pushing older ones down

üéØ What This Component Does

QuickButtonsViewModel is the rapid response control panel that dramatically speeds up repetitive inventory operations by storing the 10 most recent transactions as clickable buttons.

üí° Key Principle: This ViewModel uses event-driven communication to avoid tight coupling - it fires events when buttons are clicked, allowing other components to respond without direct dependencies.

üîÑ Advanced Operation Flow

Step 1: Auto-Discovery - System loads user's last 10 successful inventory transactions from database
Step 2: Smart Population - Buttons display Part ID, Operation number, and last used quantity
Step 3: One-Click Execution - User clicks button to instantly populate main inventory form
Step 4: Form Integration - InventoryTabViewModel receives the data and validates it
Step 5: Auto-Update - After successful operation, button list automatically updates with new transaction
Step 6: Smart Shifting - New operations move to top, older ones shift down, maintaining 10-slot limit

üè¢ MTM-Specific Business Rules

üö® Critical MTM Rule: Quick Buttons only store operation parameters (Part ID, Operation number, Quantity) but do NOT determine TransactionType - that's based on user intent when the operation is executed.

üîÑ Auto-Shifting Logic

  • New Operation: Added to position 1, all others shift down
  • Existing Operation: Quantity updated, button moves to position 1
  • Empty Slots: Filled as new operations are performed

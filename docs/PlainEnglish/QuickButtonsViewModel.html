QuickButtonsViewModel

The Quick Action Station

🎯 What This Is

The Rapid Response Inventory Control Panel

Think of QuickButtonsViewModel as your rapid response control panel - like having 10 preset buttons on a manufacturing machine that instantly repeat your most common operations. Instead of filling out forms every time, workers can simply click a button to instantly populate the inventory form with their frequently used part combinations.

🏭 Manufacturing Floor Analogy

Imagine you have a programmable control panel next to your workstation where you can:

  • ⚡ One-click operations - Click a button to instantly fill out inventory forms
  • 📋 Store your top 10 actions - Save your most frequently used part/operation combinations
  • 🔄 Auto-update from history - Buttons automatically update based on your recent work
  • ✏️ Edit button assignments - Customize what each button does via right-click menu
  • 🗑️ Remove unused buttons - Clear out actions you no longer need
  • 🔧 Manage the entire panel - Reset, clear all, or reorder buttons

⚙️ How It Works

📐 Layout Rules

  • Fixed 10 slots - Always displays positions 1 through 10
  • Recent actions at top - Most recent transactions appear in positions 1-7
  • Empty slots at bottom - Unused positions show "Empty Slot" with assignment option
  • Auto-shifting - When new actions are added, older buttons shift down
  • Right-click context menu - All management functions accessed via right-click

⚡ Available Actions

👆 Left Click - Execute Action

Instantly populates the active inventory form with the button's stored values

  • Fills Part ID field with button's part
  • Sets Operation to button's operation number
  • Enters Quantity from button's stored amount
  • Switches to Inventory tab if not already active

🖱️ Right Click - Context Menu

Opens management menu with these options:

  • Edit Button - Modify the button's stored values
  • Remove Button - Delete this button and shift others up
  • Clear All - Remove all buttons from the panel
  • Refresh - Reload buttons from recent transaction history
  • Reset Order - Restore original sequence
  • Manage Buttons - Open advanced button management

📋 MTM Business Rules

🚨 Critical Operation Rules

  • Form Population Only - Quick buttons populate forms but don't execute transactions
  • Operation Numbers - Operations (90, 100, 110) are workflow steps, NOT transaction types
  • User Context - Buttons load the last 10 transactions for the current user only
  • Real-time Updates - Button list refreshes automatically when new operations complete

⚠️ Data Management Rules

  • 10 Slot Limit - Panel always shows exactly 10 positions regardless of data
  • User-Specific Data - Each user has their own set of quick buttons
  • Automatic Shifting - When slots are full, oldest buttons are replaced by newest
  • Empty Slot Handling - Unused positions show clear "Empty Slot" indicators

ℹ️ Integration Requirements

  • Event Communication - Uses events to communicate with inventory forms
  • No Direct Database Access - All database operations via stored procedures
  • Context Menu Management - All button management through right-click menus
  • Tab Integration - Automatically switches to Inventory tab when button is clicked

🔧 Management Operations

🔄 Refresh Buttons

Reloads the quick buttons from your recent transaction history

  • Fetches your last 10 inventory operations from database
  • Updates button layout with most recent actions
  • Maintains user-specific button assignments

✏️ Edit Button

Customize what a specific button does

  • Modify Part ID, Operation, or Quantity
  • Update display text and tooltips
  • Save changes to user's button configuration

🗑️ Remove Button

Delete a button and automatically reorganize the layout

  • Removes the selected button from the panel
  • Shifts remaining buttons up to fill the gap
  • Updates position numbers automatically

🧹 Clear All

Remove all buttons and start with a clean panel

  • Clears all assigned buttons for the current user
  • Shows 10 empty slots ready for new assignments
  • Can be restored using Refresh to reload from history

🔄 Reset Order

Restore buttons to their original sequence based on transaction history

  • Reorders buttons by transaction timestamp
  • Most recent operations move to the top
  • Undoes any custom ordering changes

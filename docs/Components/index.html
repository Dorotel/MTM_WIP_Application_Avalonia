📖 Component Development Guidelines

Component documentation will be generated from markdown instruction files located in:

Development/UI_Documentation/
├── Control_QuickButtons.md
├── Control_InventoryTable.md
├── Control_TransactionHistory.md
├── Form_AddItem.md
├── Form_RemoveItem.md
├── Form_TransferItem.md
└── ...

🎯 Component Generation Process

  1. Parse markdown instruction files from Development/UI_Documentation/
  2. Extract component structure, props, and visual representation
  3. Generate Avalonia AXAML views and ReactiveUI ViewModels
  4. Create component documentation with usage examples
  5. Include business logic integration points and event patterns

🔗 Integration with Core Files

Components integrate with the following core systems:

  • Services: Business logic through dependency injection
  • ViewModels: ReactiveUI patterns for data binding
  • Models: Strongly-typed data contracts
  • Navigation: Routing and state management

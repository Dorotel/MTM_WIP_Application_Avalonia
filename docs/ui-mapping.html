📷UI Mapping Reference

⚠ Important: Before creating UI mappings, ensure theme system and navigation service are implemented for proper MVVM patterns.
This page explains how screenshots and instruction files are mapped to generate UI components. For the actual generation process, see UI Generation.
▶Mapping Process
  1. Screenshot Analysis
    • Locate the corresponding screenshot in UI_Winform_Screenshots/
    • Analyze layout, control positioning, and visual hierarchy
    • Identify modern Avalonia equivalents for WinForms controls
    • Note spacing, alignment, and grouping patterns
  2. Instruction File Review
    • Parse the .instructions.md file for component structure
    • Extract control names, types, and event definitions
    • Identify MTM-specific data patterns and relationships
    • Note context menu requirements and management features
  3. Avalonia Translation
    • Convert WinForms controls to modern Avalonia equivalents
    • Apply MTM purple brand theme using DynamicResource
    • Implement ReactiveUI binding patterns
    • Add proper MVVM separation with ViewModels

▶MTM-Specific UI Patterns

Component Management Features

  • Context Menus: Right-click functionality for Edit, Remove, Clear All, Refresh
  • Quick Actions: Keyboard shortcuts and toolbar integration
  • Space Optimization: UniformGrid for equal distribution, remove ScrollViewer when not needed
  • Data Population: Simple field population without tab switching

MTM Data Display Patterns

  • Part ID: String format display with validation highlighting
  • Operation: String numbers ("90", "100", "110") with descriptive tooltips
  • Quantity: Integer input with increment/decrement controls
  • Position: 1-based indexing for user display (Position 1, Position 2, etc.)

Button Collections (Quick Buttons)

  • Layout: UniformGrid with equal button sizes
  • Content: "(Operation) - [PartID x Quantity]" format
  • Management: Context menu for editing and removal
  • Events: Simple field population, not tab navigation
▶Modern Layout Enhancement

From WinForms Layout to Modern Avalonia

Sidebar Navigation Pattern

Convert traditional TabControl to modern sidebar:

  • Sidebar Width: 240px fixed width with navigation items
  • Content Area: Dynamic content based on selection
  • Visual Enhancement: Cards, shadows, MTM gradients

Card-Based Content

Transform grouped controls into modern cards:

  • Borders: Rounded corners (8px), subtle shadows
  • Spacing: Consistent margins and padding
  • Headers: Clear section titles with icons

Hero Sections

Add engaging banner areas:

  • Gradients: MTM purple gradient backgrounds
  • Typography: Large, bold headings with descriptions
  • Actions: Primary action buttons prominently displayed

▶Quality Verification

All UI mappings should be verified using Quality Assurance compliance reporting.

UI Mapping Compliance Check

Use this prompt to verify UI mapping compliance:

Review Views/[ViewName].axaml for compliance with UI mapping standards. Generate compliance report identifying hard-coded colors, missing DynamicResource usage, improper control mappings, and missing MTM design patterns. Include specific fixes for modern Avalonia layout enhancement.

Common Mapping Violations

  • Using WinForms control patterns instead of modern Avalonia equivalents
  • Hard-coded colors instead of DynamicResource theme integration
  • Missing context menu integration for management features
  • Improper MTM data display patterns
  • Not following modern layout enhancement guidelines

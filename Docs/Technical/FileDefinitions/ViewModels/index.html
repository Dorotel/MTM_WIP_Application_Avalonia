??? MVVM Architecture with ReactiveUI

All ViewModels in the MTM application follow the Model-View-ViewModel (MVVM) pattern using ReactiveUI for reactive programming. This ensures clean separation of concerns and testable, maintainable code.

? Reactive Properties

  • RaiseAndSetIfChanged for property updates
  • WhenAnyValue for property observation
  • Automatic UI notification

?? Commands

  • ReactiveCommand for user actions
  • Async command support
  • CanExecute reactive conditions

?? Dependency Injection

  • Constructor-based service injection
  • Transient ViewModel lifetime
  • Service layer integration
